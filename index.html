Cloud Run Functions: Qwik Start - Command Line

Setup and Configuration

  • Click on Start a Lab button
  • Go to Cloud Overview ‚Üí Check checkbox ‚Üí Click Agree
  • Open Google Cloud Console ‚Üí Copy credentials ‚Üí Sign in ‚Üí Agree
  • Click on Activate Cloud Shell ‚Üí Click Continue
  • This lab includes: Create ‚Üí Deploy ‚Üí Test function ‚Üí View logs

To connect with the project:

  • gcloud auth list ‚Üí Click Authorize
  • gcloud config list project

Task 1: Create a function

  • gcloud config set compute/region us-west1
  • mkdir gcf_hello_world && cd $_ or mkdir gcf_hello_world ‚Üí cd gcf_hello_world
  • nano index.js ‚Üí Paste the following:
const functions = require('@google-cloud/functions-framework');

functions.cloudEvent('helloPubSub', cloudEvent => {
  const base64name = cloudEvent.data.message.data;
  const name = base64name
    ? Buffer.from(base64name, 'base64').toString()
    : 'World';
  console.log(`Hello, ${name}!`);
});
  • To save and exit nano: CTRL+X, then Y, then ENTER
  • nano package.json ‚Üí Paste the following:
{
  "name": "gcf_hello_world",
  "version": "1.0.0",
  "main": "index.js",
  "scripts": {
    "start": "node index.js",
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "dependencies": {
    "@google-cloud/functions-framework": "^3.0.0"
  }
}
  • Save and exit nano
  • npm install ‚Üí Install dependencies

Task 2: Deploy your function

gcloud functions deploy nodejs-pubsub-function \
  --gen2 \
  --runtime=nodejs20 \
  --region=REGION \
  --source=. \
  --entry-point=helloPubSub \
  --trigger-topic cf-demo \
  --stage-bucket PROJECT_ID-bucket \
  --service-account cloudfunctionsa@PROJECT_ID.iam.gserviceaccount.com \
  --allow-unauthenticated
  • Wait for deployment
  • Check with: gcloud functions describe nodejs-pubsub-function --region=REGION
  • Click Check my progress

Task 3: Test the function

gcloud pubsub topics publish cf-demo --message="Cloud Function Gen2"

Task 4: View logs

gcloud functions logs read nodejs-pubsub-function --region=REGION

Task 5: Test your understanding

  • Serverless lets you write and deploy code without managing infrastructure.
  • Answer: True
  • Submit
  • End Lab

Google Kubernetes Engine: Qwik Start

Setup and Configuration

  • Click on Start a Lab button
  • Click on Cloud Overview ‚Üí Check checkbox ‚Üí Click Agree
  • Open Google Cloud Console ‚Üí Copy credentials ‚Üí Sign in ‚Üí Agree
  • Click on Activate Cloud Shell ‚Üí Click Continue
  • In this lab: Create a Kubernetes cluster, Deploy & expose app, View logs

To connect with the project:

  • gcloud auth list ‚Üí Click Authorize
  • gcloud config list project

Task 1: Set a default compute zone

  • gcloud config set compute/region us-west1 or gcloud config set compute/region "REGION"
  • gcloud config set compute/zone us-west1-c or gcloud config set compute/zone "ZONE"

Task 2: Create a GKE cluster

gcloud container clusters create --machine-type=e2-medium --zone=us-west1-c lab-cluster
# or
# gcloud container clusters create --machine-type=e2-medium --zone=ZONE lab-cluster
  • Wait for the cluster to be created
  • Click Check my progress

Task 3: Get authentication credentials for the cluster

gcloud container clusters get-credentials lab-cluster

Task 4: Deploy an application to the cluster

  • kubectl create deployment hello-server --image=gcr.io/google-samples/hello-app:1.0
  • Click Check my progress
  • kubectl expose deployment hello-server --type=LoadBalancer --port 8080
  • kubectl get service
  • Wait for EXTERNAL IP ‚Üí Open in browser: http://[EXTERNAL-IP]:8080
  • Click Check my progress

Task 5: Delete the cluster

gcloud container clusters delete lab-cluster
  • Wait for deletion
  • Click Check my progress
  • Submit and end the lab
